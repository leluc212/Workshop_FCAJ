Advanced Analytics with Athena and AI-Powered Serverless Security Analytics System on AWS

I. Giới thiệu tổng quan hệ thống

Hệ thống được xây dựng nhằm phân tích log từ AWS WAF để phát hiện và đánh giá các hành vi tấn công web một cách hiệu quả và tiết kiệm chi phí. Dữ liệu log được lưu trữ trên Amazon S3 dưới dạng raw, sau đó được chuyển đổi sang định dạng tối ưu (Parquet) nhằm giảm chi phí truy vấn bằng Amazon Athena.

Hệ thống áp dụng mô hình phân tích hai lớp:

  • Lớp thứ nhất sử dụng Athena để lọc các request đáng nghi dựa trên rule và thống kê
  • Lớp thứ hai sử dụng Amazon Bedrock để phân tích nâng cao, giúp giảm false positive

Ngoài ra, hệ thống sử dụng cơ chế event-driven và threshold-based AI invocation để đảm bảo chỉ xử lý khi cần thiết, từ đó tối ưu chi phí vận hành.

II. Kiến trúc tổng thể

Hệ thống được thiết kế theo mô hình serverless và chia thành các lớp:

  • Edge & Security Layer
  • Processing & Cataloging Layer
  • Analysis & Reporting Layer
  • Report & Notification Layer

Kiến trúc hướng đến:

  • Tối ưu chi phí truy vấn
  • Giảm thiểu xử lý không cần thiết
  • Tăng độ chính xác thông qua AI

III. Edge and Security Layer

Lớp này bao gồm Amazon CloudFront và AWS WAF.

  • CloudFront tiếp nhận request từ người dùng và phân phối nội dung
  • AWS WAF kiểm tra request dựa trên rule bảo mật (SQLi, XSS, rate limit...)
  • Toàn bộ log từ WAF được ghi lại và lưu trữ để phục vụ phân tích

IV. Processing and Cataloging Layer

Lưu trữ dữ liệu

Log từ WAF được lưu vào:

  • S3_rawlog (JSON – dữ liệu gốc)

Tối ưu dữ liệu

AWS Lambda được kích hoạt bởi S3 event để:

  • Chuyển đổi JSON → Parquet
  • Partition dữ liệu theo thời gian

Dữ liệu sau đó được lưu vào: S3_parquet

Catalog dữ liệu

AWS Glue Crawler được sử dụng để:

  • Scan dữ liệu từ S3_parquet
  • Tạo schema trong AWS Glue Data Catalog

Data Catalog đóng vai trò metadata repository giúp Athena truy vấn dữ liệu.

Truy vấn dữ liệu

Amazon Athena:

  • Truy vấn trực tiếp trên S3_parquet
  • Lọc các request đáng nghi (BLOCK, spike, abnormal pattern)

V. Event-driven Query Mechanism

Thay vì sử dụng cơ chế schedule, hệ thống sử dụng:

  • Trigger từ S3 hoặc logic trong Lambda

Lambda athena-query:

  • Gửi query đến Athena
  • Lấy kết quả đã filter

VI. Analysis and Reporting Layer

Data Preprocessing

AWS Lambda (data-preprocessor):

  • Chuẩn hóa dữ liệu từ Athena
  • Tổng hợp thông tin (IP, request count, pattern...)

Threshold-based AI Invocation

Hệ thống áp dụng cơ chế:

  • Chỉ gọi Amazon Bedrock khi dữ liệu vượt ngưỡng nghi ngờ

AI Analysis

Amazon Bedrock:

  • Phân tích hành vi request
  • Phân loại:
    • True attack
    • False positive
  • Trả về:
    • Risk level
    • Explanation

Decision Layer

AWS Lambda decision-layer:

  • Nhận kết quả từ Bedrock
  • Quyết định:
    • Alert
    • Lưu log
    • Bỏ qua

Lưu kết quả: Kết quả được lưu vào S3_AI_Report phục vụ cho việc báo cáo.

VII. Report and Notification Layer

AWS Lambda report-delivery:

  • Tổng hợp báo cáo từ S3_AI_Report

Amazon SNS:

  • Gửi cảnh báo đến admin
  • Trigger khi: Phát hiện attack nghiêm trọng

VIII. Lợi ích của hệ thống

Hệ thống mang lại các lợi ích:

Tối ưu chi phí

  • Athena chỉ tính phí theo data scan
  • Parquet giảm đáng kể chi phí query
  • AI chỉ chạy khi cần (threshold-based)

Hiệu năng cao

  • Query nhanh hơn nhờ columnar format
  • Event-driven giảm latency

Độ chính xác cao

  • Kết hợp rule-based + AI
  • Giảm false positive

Kiến trúc serverless

  • Không cần quản lý server
  • Tự động scale

IX. Dọn dẹp tài nguyên

Xóa tài nguyên AWS Glue

  • Xóa các ETL jobs đã tạo cho quá trình xử lý dữ liệu
  • Xóa workflows và triggers liên quan đến Glue jobs
  • Xóa toàn bộ crawlers dùng để thu thập schema
  • Xóa databases và tables trong Glue Data Catalog để loại bỏ metadata

Xóa dữ liệu trên S3

  • Xóa toàn bộ dữ liệu trong các bucket S3 bao gồm S3_rawlog và S3_AI_Report
  • Xóa các bucket sau khi đã đảm bảo không còn dữ liệu quan trọng

Xóa tài nguyên Athena

  • Xóa các query đã lưu và file kết quả truy vấn trên S3
  • Xóa các database và table không còn sử dụng trong Athena

Xóa AWS Lambda

  • Xóa tất cả các Lambda function đã sử dụng trong hệ thống như preprocess, decision và report
  • Kiểm tra và đảm bảo không còn trigger liên kết với các function đã xóa

Xóa SNS

  • Xóa các SNS topic và toàn bộ subscription liên quan
  • Đảm bảo không còn gửi thông báo sau khi dọn dẹp

Xóa IAM

  • Xóa các IAM role đã tạo cho Lambda, Firehose và Glue
  • Xóa các custom policy đi kèm
  • Kiểm tra và dọn dẹp cấu hình lưu log trên CloudWatch

X. Chi phí, thời gian ước tính

Thời gian ước tính: 6h - 8h

Chi phí ước tính: 3$ - 8$

XI. Kết luận

Hệ thống đề xuất là một giải pháp phân tích bảo mật hiện đại, tận dụng kiến trúc serverless trên AWS kết hợp với trí tuệ nhân tạo.

Việc bổ sung các cơ chế:

  • Data optimization (Parquet)
  • Event-driven processing
  • Conditional AI invocation

giúp hệ thống đạt được:

  • Hiệu quả cao
  • Chi phí thấp
  • Khả năng mở rộng tốt
